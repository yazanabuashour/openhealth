# OpenHealth v0.2.2

This patch release keeps the client install story focused on the supported production path: install the released `openhealth` runner, verify it with `openhealth --version`, then register the OpenHealth skill with the agent's native skill system.

## Changed

- Added an agent-directed upgrade prompt to the README that mirrors the install prompt.
- Simplified manual upgrade guidance to use the GitHub Release installer.
- Removed client-facing references to development-tool install paths.
- Added hand-authored release notes support for future GitHub Releases.

## Verification

- Runner behavior and skill task behavior are unchanged from v0.2.1.
- No production eval rerun is required for this docs and release-workflow patch.
