# OpenHealth v0.3.0

This feature release expands OpenHealth import coverage so agents can preserve narrative context that previously had to be skipped while keeping the local runner and single-file skill model unchanged.

## Changed

- Added top-level notes for weight and blood-pressure records.
- Added ordered per-result notes for lab results and imaging records so result-specific narrative text stays attached to the measured row.
- Preserved multiline note text and readback/export behavior across create, list, correction, and patch flows.
- Updated the OpenHealth skill so agents store narrative context in the narrowest matching note field.

## Verification

- The production runner/skill eval passed all 50 production scenarios with stop-loss false and recommendation `ship_openhealth_runner_production`.
- Release assets, checksums, attestations, the single-file skill archive, and installer smoke tests were verified after publication.
