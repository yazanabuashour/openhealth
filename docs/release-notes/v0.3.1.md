# OpenHealth v0.3.1

This patch release tightens the production OpenHealth skill so agents spend less effort on routine validation and readback while preserving the same local runner, storage, and Go package contracts.

## Changed

- Made rule-covered invalid requests direct-reject cases so agents do not call runner validation or inspect files before refusing unsupported input.
- Clarified that same-domain rows should be batched into one runner request and that write responses can be used directly when they already contain the requested final rows.
- Tightened active medication, bounded range, and lab correction guidance so agents avoid mentioning omitted rows, use explicit `YYYY-MM-DD` dates, avoid local path discovery, and treat `correct_labs` as full collection replacement.

## Verification

- The production runner/skill eval passed all 50 production scenarios with stop-loss false and recommendation `ship_openhealth_runner_production`.
- The focused `oh-cpe` eval showed direct-reject scenarios using zero tools and reduced avoidable follow-up reads in the targeted production scenarios.
