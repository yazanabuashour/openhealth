# OpenHealth v0.4.0

This feature release adds subjective sleep tracking to the production OpenHealth runner and skill while keeping the same local-first JSON runner model and direct-local Go package contract.

## Changed

- Added sleep check-ins with wake-date, subjective 1-5 quality, optional wakeup count, optional note, latest/history/range listing, same-date upsert correction, delete support, and invalid-input rejection.
- Updated the production skill so sleep tasks use the installed runner directly and report date-first rows from returned entries.
- Added release-gate coverage for sleep scenarios in the production `oh-5yr` agent eval suite.
- Added ongoing security operations and maintainer-isolation documentation for recurring dependency review, advisory rehearsal, threat-model refreshes, untrusted pull request policy, and automation least privilege.
- Tightened production eval verification for narrative-note punctuation and sleep final-answer recognition without changing the runner JSON contract.
