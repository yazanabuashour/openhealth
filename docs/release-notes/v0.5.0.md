# OpenHealth v0.5.0

This minor pre-1.0 release retires the legacy data-directory configuration path, standardizes explicit local storage configuration on database file paths, and hardens production eval isolation while keeping the same runner JSON domains and release asset contract.

## Changed

- Retired `OPENHEALTH_DATA_DIR`, `client.EnvDataDir`, and `LocalConfig.DataDir`; explicit storage overrides now use `OPENHEALTH_DATABASE_PATH` or `client.LocalConfig{DatabasePath: "..."}`.
- Added local SQLite-backed OpenHealth config storage with migration 0008 and repository get/list/upsert/delete helpers for future runtime settings while keeping database-file discovery outside database-stored config.
- Updated the production skill and public docs to refer to the configured local database path instead of the retired data-directory path.
- Removed stale examples and updated development commands to exercise the production runner JSON path with repo-pinned `mise` tooling.
- Hardened the `oh-5yr` production eval harness so runs use an isolated auth-only `CODEX_HOME`, pass `--ignore-user-config`, and report neutral `<run-root>` artifact references.
- Updated release attestation actions, the SQLite dependency, local golangci-lint configuration, and macOS metadata ignore rules.

## Verification

- The production runner/skill eval passed all 53 production scenarios with stop-loss false and recommendation `ship_openhealth_runner_production`.
- The v0.5.0 final eval reported zero direct generated-file inspection, broad repo search, generated paths from broad search, module-cache inspection, retired CLI usage, or direct SQLite access.
- Eval history isolation passed with 50 single-turn ephemeral runs and 3 persisted multi-turn sessions under an isolated `<run-root>/codex-home`.
- Local release gates passed: release-doc validation, skill validation, generated-code drift check, formatting check, golangci-lint, unit tests, and committed-doc absolute-path policy grep.
